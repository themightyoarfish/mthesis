 export : Package

Package used by model definition code to inform ikkuna of all the things that need to be exported to visualization clients, such as

 

  • graph structure, layer names, sizes, ops
  • unit activations
  • unit gradients
  • unit weights
  • model predictions

 

Appropriate asynchronous callbacks into the introspection package will be added to the graph execution.

 definitions : Package

This package contains base classes for metadata, weights, activations, gradients and other stuff so that key-value maps can be exported from the model which are more easily serialized and worked with.

 serialize : Package

Use the data exported in formats defined in the definitions package to add summary variables (tf.summary or tensorboardX.summary) for serialization.

 export package usage : Note

Use either of the subpackages to create definitions and register them for introspection.

 introspection : Package

Plugin framework for consuming the data in the formats defined by the definitions package. Arbitrary metrics can then be computed from the live data and passed to the serialize package for writing to disk.

  : Note

Distill live data into meaningful metrics.

 visualization : Package

This package contains the backend (python) portions for the tensorboard visualizations.
